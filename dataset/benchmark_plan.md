# Benchmark v0 (кратко)

## Зачем нужен бенчмарк
Бенчмарк — фиксированный набор тестовых кейсов (вход → ожидаемый результат), необходимый для воспроизводимой оценки качества интеллектуального ассистента. Он позволяет измерять качество системы не “на глаз”, а по заранее заданным задачам и метрикам.

## Связь с датасетом
Датасет (NVD + CISA KEV) используется как источник фактов и описаний уязвимостей.  
Бенчмарк формируется поверх датасета и задаёт тестовые сценарии, на которых оценивается работа ассистента.

## Формат
Benchmark v0 хранится в JSONL: одна строка = один тестовый кейс. Это упрощает автоматическую обработку и масштабирование.

## Типы задач в Benchmark v0
1) **CWE-классификация**: по описанию CVE определить CWE.  
2) **Суммаризация**: кратко описать суть уязвимости (1–2 предложения).

## Как будет оцениваться качество
- Для CWE-классификации: Accuracy / Macro-F1.
- Для суммаризации: экспертная оценка (v0), далее возможно ROUGE/BERTScore и/или автоматизированная оценка.
- В дальнейшем оценка будет разделяться на retrieval-метрики (качество поиска контекста) и generation-метрики (качество ответа).
